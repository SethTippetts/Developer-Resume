About me

As a developer...

I love learning new things! I typically would take depth of knowledge over breadth. If I'm gonna take the time to learn something new, I wanna be an expert!

Not a big fan of wasting time. If I find myself in a meeting I don't need to be in, or we're discussing something we already decided on, I'm the first person to cordially excuse myself. Development is too fun to waste time!

I'm pretty opinionated when it comes to code. I like to challenge peoples ideas and hope they challenge mine. I like bringing other devs up to speed on a new product or technology and watch them shine.

My most recent kick is explicit programming. I've had problems in longer living projects with implicit magic code that does things I don't always intend or remember. This largly includes dynamic module loading (i.e. `swaggerize-express`, `config`, etc) It's difficult to walk the line between DRY code and explicit code. I think some duplication is worth knowing how something works, especially long term.

I value flexibility and a company that knows some of the best code is written in a 2am coffee fueled frenzy, not at exactly 9am.
